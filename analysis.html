3. Estrategia de Nombrado de Recursos (Resource Naming Strategy)

3.1 Esquema General de URIs

Base URI del proyecto: http://linkeddata.greenzonesmadrid.org/

Patrón general: {base-uri}/{tipo}/{identificador}

3.2 URIs para la Ontología

Namespace: http://linkeddata.greenzonesmadrid.org/ontology/gz#

Prefix: gz:

Ejemplos:

  • Clase GreenZone: gz:GreenZone
  • Clase District: gz:District
  • Propiedad inDistrict: gz:inDistrict
  • Propiedad fileNumber: gz:fileNumber

3.3 URIs para Recursos (Instancias)

Zonas Verdes

Patrón: http://linkeddata.greenzonesmadrid.org/resource/zone/{id}

Identificador: Número secuencial basado en el número de archivo del dataset original

Ejemplos:

  • http://linkeddata.greenzonesmadrid.org/resource/zone/1
  • http://linkeddata.greenzonesmadrid.org/resource/zone/2
  • http://linkeddata.greenzonesmadrid.org/resource/zone/100

Distritos

Patrón: http://linkeddata.greenzonesmadrid.org/resource/district/{NOMBRE}

Identificador: Nombre del distrito en mayúsculas (tal como aparece en el dataset)

Ejemplos:

  • http://linkeddata.greenzonesmadrid.org/resource/district/CENTRO
  • http://linkeddata.greenzonesmadrid.org/resource/district/RETIRO
  • http://linkeddata.greenzonesmadrid.org/resource/district/CHAMBERI

3.4 Principios de Diseño

  • Persistencia: Las URIs están diseñadas para ser permanentes y no cambiar
  • Legibilidad: Uso de nombres descriptivos en inglés para alcance internacional
  • Jerarquía clara: Separación entre ontología (/ontology/) y recursos (/resource/)
  • Consistencia: Uso sistemático de minúsculas en paths, mayúsculas solo en identificadores de distrito
  • Escalabilidad: IDs numéricos permiten añadir nuevas zonas fácilmente
  • Desreferenciabilidad: Aunque no implementado, las URIs siguen las mejores prácticas para poder servir información en el futuro

3.5 Enlaces Externos

Estrategia de linking: Uso de owl:sameAs para conectar con Wikidata

Ejemplo:

  • Distrito CENTRO enlazado a: https://www.wikidata.org/entity/Q1763376

3.6 Convenciones de Nombrado de Propiedades

  • camelCase: Para propiedades en inglés (ej: fileNumber, inDistrict)
  • lowercase: Para términos simples (ej: district, plot)
  • Multilingüismo: Labels en español e inglés mediante rdfs:label

6. Notas Adicionales

  • El dataset contiene 648 registros de zonas verdes en Madrid
  • Todos los registros pertenecen a la categoría "ZONAS VERDES" y "PARQUES Y JARDINES"
  • La transformación a RDF incluye enlaces a Wikidata para facilitar la integración con otras fuentes de datos
  • Se utiliza una ontología ligera personalizada (gz:) para modelar los datos
